🎙 Voice-to-Text Workflow
🎙️ Write Prompts Without Typing — Voice-to-Text Workflow
Writing long AI prompts can feel difficult, especially on a phone. The solution is voice-to-text. Here is the best way to do it — and why one method is strongly preferred over another.
⭐ Pro Technique

Speak Your Prompt — Then Paste It into Claude

Instead of typing your prompt from scratch, simply speak it aloud into a voice-to-text tool. The tool converts your words into text. You copy that text and paste it into Claude. This is how professionals generate detailed prompts quickly — especially on a phone.

The key rule: speak in clear sentences as if you are dictating a letter. Tell the AI exactly what you want it to type, and say "type this word for word, no edits, no paraphrasing."

✅ Recommended
💬
Meta AI on WhatsApp
Free · Works on any Android phone
  1. 1
    Open WhatsApp. Tap the Meta AI chat (the circle icon, or search @Meta AI).
  2. 2
    Hold the microphone button on your keyboard and speak your prompt clearly. Keep each voice note under 5 minutes.
  3. 3
    Before sending, say: "Please type out everything I just said, word for word, with no edits and no paraphrasing."
  4. 4
    Meta AI will type out your exact words as text.
  5. 5
    Copy the text. Open Claude (claude.ai). Paste and send.
✅ Why WhatsApp? Your voice note is saved to WhatsApp even if you lose internet midway through recording. You can finish speaking, reconnect, and your audio is still there. Nothing is lost.
🤖
Claude Voice (claude.ai)
Available in the Claude app
  1. 1
    Open claude.ai in your browser or the Claude app.
  2. 2
    Tap the microphone icon in the message box.
  3. 3
    Speak your prompt clearly. Claude transcribes it live.
  4. 4
    Review the text and send.
⚠️ Network risk: If you lose internet while speaking in Claude, your voice recording is lost. Claude's voice input requires a continuous connection. If your network is intermittent, use Meta AI on WhatsApp instead.
⏱️
Keep each voice note under 5 minutes. If your prompt is long, break it into parts. Record Part 1, get the text, then record Part 2. Combine both texts in Claude. Meta AI handles multiple voice notes in one conversation perfectly.
Example — what to say into WhatsApp
🎤 You say this (spoken aloud):
"Meta AI, please type out exactly what I say now, word for word, no edits. Write me a complete Python Flask server that connects to a PostgreSQL database. It should have endpoints for student registration, login with session token, password reset, and a dashboard. Use environment variables for the database URL and JWT secret. Include full error handling. End of prompt."
🤖 Claude Features You Should Know
🤖 Claude Features That Will Help You Build Faster
Claude is not just a chat window. It has powerful features built specifically for developers and creators working on projects like this one.
📁
Claude Projects
Claude Projects lets you create a dedicated workspace for your training portal. Upload your code files, database schema, and any documents. Claude will remember the context of your entire project across all your conversations — no need to re-explain every time.
💡 Create one Project called "Training Portal" and upload your app.py, requirements.txt, and any notes. Every conversation inside that project has full context.
🔌
Claude API
The Claude API is what powers the automatic assignment marking we build in Step 12. It lets your server send text to Claude and receive a structured response — scores, grades, and feedback — automatically. No human clicks required.
💡 The API costs approximately $3 per million input tokens. Marking 300 student essays costs roughly ₦150–₦400 total.
🎤
Claude Voice Input
Claude supports voice input on mobile. But remember: if you lose internet while speaking, your recording is lost. For reliable voice-to-text, use Meta AI on WhatsApp first, then paste the text into Claude. See the Voice-to-Text section above.
💡 Claude voice works best in a strong Wi-Fi environment. For fieldwork or intermittent data, always use WhatsApp + Meta AI.
📤
Prompt Pasting Workflow
The most efficient workflow: speak into WhatsApp → get text from Meta AI → copy → paste into Claude. Claude has no character limit on pasted prompts. You can paste entire specifications, requirements documents, or lengthy instructions and Claude will follow them precisely.
💡 You can also paste code Claude gave you, add "now add payment integration to this," and Claude will update the exact code you gave it.

⚙ Three Pillars of Low-Data AI
⚙️ The Three Pillars of Low-Data AI
Before we build, you need to understand why this is possible with very little data. These three techniques make it work.
01
Transfer Learning
Borrow a brain someone else already trained on millions of examples. Adapt it to your smaller, specific task in days — not years. Like hiring an expert doctor and teaching them your clinic's system.
02
Prompt Engineering
Get maximum intelligence from existing AI by writing precise instructions. A great prompt can replace months of model training. No code required. No data required. Just the right words.
03
Synthetic Data
When you don't have enough real data — generate it with AI. Especially powerful in Africa where large labelled datasets for our languages, crops, and diseases rarely exist.
💡
Key insight: Together these three techniques mean you can build functional AI systems without a supercomputer, without millions of naira, and without a PhD. Everything in this workshop uses free or near-free tools.
Module 1 — GitHub Setup
1
Create Your GitHub Account
~5 mins

GitHub is the home of your code and your free website hosting. Think of it as Google Drive, but for code. Everything you build goes here first.

ℹ️
Sign up with Gmail. Use your Gmail account to create GitHub. This keeps everything connected and means fewer passwords to remember.
  1. Open your browser and go to github.com. Press Enter.
  2. Click Sign up in the top right corner.
  3. On the sign-up page, look for the option:
    G
    Continue with Google
    Click it. Sign in with your Gmail account.
  4. GitHub will ask you to confirm your Google account. Click Continue.
  5. GitHub will ask "How many people will be using GitHub?" Select Just me.
  6. It may ask your role. Select Teacher or Student to unlock free premium features for educators.[1]
  7. Select the Free plan. Click Continue for free.
  8. You are now inside GitHub. ✅
⚠️
Username tip: GitHub will create a username from your Gmail name. You can change it. Choose something professional — it will appear in your website URL. Example: rain-institute → becomes rain-institute.github.io. To change: click your profile picture → Settings → change username.
2
Create Your Special Repository
~5 mins
ℹ️
Two types of GitHub site — pick one:

Option A — User Site (1 per account): Name the repo exactly YOURUSERNAME.github.io. Your site lives at https://YOURUSERNAME.github.io. You only get one of these.

Option B — Project Site (unlimited): Name the repo anything, e.g. ai-training. Your site lives at https://YOURUSERNAME.github.io/ai-training. You can have as many as you like.

✅ For a custom domain (like rain-institute.ng), both options work equally well. Choose Option A for your main institution website, Option B for sub-projects.
🔴
Critical for Option A: The repository name must be exactly YOURUSERNAME.github.io. Type it wrong and GitHub Pages will not activate automatically.
  1. Inside GitHub, click the + icon in the top-right corner.
  2. Click New repository from the dropdown.
  3. In the Repository name box, type your chosen name (see options above).
  4. You should see a green message: "✓ [name] is available."
  5. Select Public. Required for free GitHub Pages hosting.
  6. Tick Add a README file.
  7. Click Create repository. ✅
3
Enable GitHub Pages — Your Free Website
~3 mins

GitHub Pages turns your repository into a live website — automatically, for free, forever.

  1. Inside your repository, click the Settings tab (top row of tabs).
  2. In the left sidebar, scroll down and click Pages.
  3. Under Build and deployment → Source, select Deploy from a branch.
  4. Under Branch, select main and / (root). Click Save.
  5. Wait 1–2 minutes. A green box will appear: "Your site is live at https://YOURUSERNAME.github.io"
  6. Click that link. Your website is live! 🌍
💡
If the green box doesn't appear immediately, wait 2 full minutes and refresh the page. Do not click Save multiple times.
Module 2 — Building Your Website
4
Build Your Training Portal Using AI
~15 mins

You will speak or type your requirements to Claude, get the HTML code, and upload it to GitHub. Your website is live within minutes.

🎙️
Use voice-to-text here! This prompt is long. Instead of typing it, speak it into WhatsApp → get the text from Meta AI → copy → paste into Claude. See the Voice-to-Text section above for full instructions.
📋 Paste this into Claude (after editing your institute name)
Write me a complete, professional HTML page for a training institute called [REPLACE WITH YOUR INSTITUTE NAME]. Include: a navigation bar with links to Home, Courses, About, and Contact. A large hero section with a headline and an "Enroll Now" button. A section showing three course cards — AI Fundamentals, Data Science, and Robotics. A footer with contact information. Put all CSS inside the same file. Make it fully responsive for mobile phones. Use a professional navy blue and gold colour scheme. Include hover effects on buttons and cards. Make it look like a real Nigerian educational institution.
  1. Go to claude.ai. Log in with your Gmail account:
    G
    Continue with Google
  2. Paste the prompt above (with your institute name). Press Enter.
  3. Claude generates HTML code. Click Copy at the top of the code block.
  4. Go back to GitHub → your repository → click Add file → Create new file.
  5. Name the file: index.html
  6. Paste the code into the text area. Click Commit changes → Commit changes again.
  7. Wait 1–2 minutes. Visit https://YOURUSERNAME.github.io to see your website live! ✅

Part A — Add DNS Records to Your Domain Registrar

You must add all four A records plus the CNAME. Adding all four A records gives your site redundancy — if one of GitHub's servers goes down, traffic automatically moves to another.[2]

TypeHost / NameValue
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153
CNAMEwwwYOURUSERNAME.github.io
  1. Buy your domain from any registrar above. Log into its dashboard.
  2. Find DNS Management or Manage DNS for your domain.
  3. Add all four A records (one at a time) and the CNAME record exactly as shown in the table above. Save each one.

Part B — Link the Domain in GitHub

  1. In GitHub → your repository → Settings → Pages.
  2. Under Custom domain, type your domain name (e.g. rain-institute.ng). Click Save.
  3. Once GitHub verifies it, tick Enforce HTTPS. This gives your site the padlock security icon.

Part C — Create the CNAME File in Your Repository

🔴
Important: You must create a file named CNAME (capital letters, no extension) inside your repository. Without this file, your custom domain link breaks every time you redeploy or push new code.
  1. In your repository, click Add file → Create new file.
  2. In the filename box, type exactly: CNAME (all capitals, no .txt or any extension).
  3. In the content area, type only your domain name on one line, e.g: rain-institute.ng
  4. Click Commit changes. Your domain link is now permanently saved. ✅
⏱️
DNS propagation time: After adding DNS records, your domain may take anywhere from 30 minutes to 24 hours to go fully live — depending on your registrar and internet providers. This is normal. Do not panic if it is not immediate. Check again the next morning if needed.
Module 3 — Backend Server
6
Create Your Render Account — Free Server
~5 mins

Render hosts your server for free and automatically re-deploys it every time you update your code on GitHub.[3]

  1. Go to render.com.
  2. Click Get Started for Free.
  3. Click Continue with Google. Sign in with your Gmail. This connects Render to your Google account — no new password needed.
  4. Render may ask to also connect your GitHub. Click Connect GitHub. Authorise it.
  5. You are inside the Render dashboard. It will look empty — that is correct. ✅
⏱️
Free tier note: On Render's free plan, your server sleeps after 15 minutes of no activity. The first request after sleep takes ~30 seconds to respond. For a live training institute with regular use, upgrade to Starter at $7/month to keep it always-on.
7
Connect Render to GitHub — Auto-Deploy
~10 mins

Once connected, every code push to GitHub automatically updates your live server. No manual uploading ever again.

⚠️
Before this step: Create a second GitHub repository called training-portal-backend. Set it to Private. This is where your server code will live — separate from your public website.
  1. In Render dashboard, click New + → Web Service.
  2. Under "Connect a repository," find training-portal-backend. Click Connect.
  3. Name: type training-portal-api
  4. Region: select Frankfurt (EU Central) — closest to Nigeria for fastest response times.
  5. Environment: select Python 3
  6. Build Command: type pip install -r requirements.txt
  7. Start Command: type python app.py
  8. Scroll down, select Free plan.
  9. Click Create Web Service. Deployment takes 2–5 minutes the first time.
  10. Once live, you see a green "Live" badge and a URL like training-portal-api.onrender.com. ✅
🔴
Environment Variables (Secrets): Never put passwords in your code. In Render → your service → left sidebar → Environment → Add Environment Variable for each secret. We will add these as we go.
8
Set Up Your Database with Neon
~8 mins

Neon provides a free PostgreSQL database where all your student records, assignments, and grades are stored.[4]

  1. Go to neon.tech. Click Sign Up Free.
  2. Click Continue with Google. Sign in with Gmail.
  3. Project name: type training-portal. Click Create project.
  4. Neon shows a Connection string starting with postgresql://. Click Copy.
  5. In Render → your service → Environment → Add Environment Variable:
  6. Key: DATABASE_URL · Value: paste the connection string. Click Save Changes. ✅
Module 4 — Services & AI Marking
9
Add Email with Mailtrap
~5 mins
  1. Go to mailtrap.io. Click Sign Up Free.
  2. Click Sign up with Google. Use Gmail.
  3. After logging in: left sidebar → Email Sending → SMTP/API Settings.
  4. You will see: Host, Port, Username, Password. Copy each one.
  5. In Render → Environment, add these four variables:
    MAIL_HOST = smtp.mailtrap.io MAIL_PORT = 587 MAIL_USER = (your Mailtrap username) MAIL_PASS = (your Mailtrap password)
  6. Click Save Changes. ✅
10
Build Your Server Using AI Prompt Engineering
~20 mins

You will speak or type this prompt to Claude, get the Python server code, and upload it to GitHub. Render auto-deploys it.

🎙️
Speak this prompt! This is a long prompt — perfect for voice-to-text. Open WhatsApp → Meta AI → hold microphone → speak the prompt below. Tell Meta AI: "type this word for word, no edits." Then paste the text into Claude.
📋 Server Code Prompt — Speak or Paste into Claude
Write me a complete Python Flask web server for a training management system. Use PostgreSQL with psycopg2. The server must have these endpoints: POST /api/register — accepts email, password, full name. Creates a student account. Sends a welcome email. POST /api/login — accepts email and password. Returns a JWT token if valid. POST /api/forgot-password — accepts email. Generates a reset token and sends a reset email. GET /api/dashboard — requires JWT token in the Authorization header. Returns student profile and enrolled courses. GET /api/courses — returns all available courses as JSON. Use environment variables for DATABASE_URL, JWT_SECRET, MAIL_HOST, MAIL_PORT, MAIL_USER, and MAIL_PASS. Include full error handling. Run on port 5000. Also give me a requirements dot txt file listing all dependencies.
  1. Paste the prompt into Claude. Claude generates two files: app.py and requirements.txt.
  2. In your training-portal-backend GitHub repository: click Add file → Create new file.
  3. Name it app.py. Paste the server code. Click Commit changes.
  4. Create another file. Name it requirements.txt. Paste that content. Click Commit changes.
  5. In Render → Environment → Add: JWT_SECRET = any long random string of letters and numbers. Click Save Changes.
  6. Render auto-detects the new code and re-deploys. Watch the Logs tab — look for "Server running on port 5000". ✅
11
Add Nigerian Payments with Paystack
~10 mins

Paystack accepts cards, bank transfers, USSD, and mobile money. A personal Gmail account is sufficient to start receiving payments.[5]

  1. Go to paystack.com. Click Create a Free Account.
  2. Sign up using your Gmail address. Verify your email.
  3. Inside the Paystack dashboard: Settings → API Keys & Webhooks.
  4. Copy your Test Secret Key (starts with sk_test_).
  5. In Render → Environment: Key PAYSTACK_SECRET_KEY · Value: paste your key. Click Save Changes.
  6. Ask Claude to add payment to your server: "Add a Paystack payment endpoint POST /api/enroll to my Flask server. Accept course_id and JWT token. Create a Paystack transaction for the course fee. Return the payment URL."
12
🤖 AI Assignment Marking with Claude API
~20 mins

This is the centrepiece. A student submits an essay. Your server sends it to Claude with a grading rubric. Claude returns a score and written feedback in seconds. 300 essays marked in under 10 minutes.

  1. Go to console.anthropic.com. Click Continue with Google. Sign in with Gmail.
  2. Click API Keys in the left sidebar. Click Create Key.
  3. Name your key training-portal. Copy the key immediately — it starts with sk-ant-. Save it — Anthropic shows it only once.
  4. In Render → Environment: Key CLAUDE_API_KEY · Value: paste your key. Click Save Changes.
📋 Ask Claude to add AI marking to your server
Add an assignment marking endpoint to my Flask server. POST /api/submit-assignment — accepts assignment_id, student JWT token, and essay_text. Call the Anthropic Claude API using model claude-sonnet-4-20250514 with this system prompt: You are a university lecturer at a Nigerian institution. Mark this student essay fairly. Return ONLY valid JSON with fields: content_score (0 to 10), argument_score (0 to 10), language_score (0 to 10), total_score (sum out of 30), grade (A B C D or F), and feedback (two to three sentences of constructive comments for the student directly. Save the result to the database. Return the score and feedback to the student. Handle API errors gracefully.
💰
Claude API cost: Marking 300 student essays costs approximately ₦150–₦400 total. Significantly cheaper than any human marking arrangement at scale.[6]
