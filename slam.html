What is SLAM?

SLAM — Simultaneous Localization and Mapping — is the process by which a robot builds a map of an unknown environment while at the same time figuring out its own position within that map. These two tasks are deeply interdependent: a good map requires knowing where you are, but knowing where you are requires a good map.

🎯 The Core Challenge
Imagine being blindfolded and dropped in an unknown room. You explore by feel, remember what you've touched, and build a mental floorplan — all while tracking your own movements. SLAM gives robots exactly this capability, using sensors instead of touch.

1.1 Odometry and Our Angle Convention

Odometry estimates a robot's position and orientation by measuring how far its wheels have turned. The key convention used throughout these notes:

⚠ Critical Convention — Vertical Axis (y-axis)
The heading angle θ is measured from the y-axis (vertical), not the x-axis. This matches compass conventions — θ = 0 means facing straight up (+y). When θ = 0 and the robot moves forward, y increases and x stays constant.

A wheel encoder counts rotations of the wheel shaft. Each count is a tick. Two common types:

distance = 1250 × (π × 0.065) / 500 = 0.5105 m ≈ 51 cm

1.3 Deriving the Kinematic Equations from Geometry

Step A — The Instantaneous Center of Curvature (ICC)

When the two wheels travel different distances in the same interval, the robot moves along a circular arc. Both wheels share the same rigid chassis, so they must rotate about a common center — the ICC.

ICC * /| / | R + L/2 (outer, right wheel) dR→ / | / | [LW]─────[Center]─────[RW] \ | \ | R − L/2 (inner, left wheel) \ | \|

Let R = turning radius of robot center, L = wheelbase, Δθ = angle swept. Both wheels trace arcs through the same angle Δθ:

dL = (R − L/2) × Δθ      ...[1] (left wheel, inner)
dR = (R + L/2) × Δθ      ...[2] (right wheel, outer)

Step B — Deriving Δθ (Heading Change)

Subtract equation [1] from equation [2]:

dR − dL = (R + L/2)×Δθ − (R − L/2)×Δθ
         = Δθ × [(R + L/2) − (R − L/2)]
         = Δθ × L

Δθ = (dR − dL) / L

✓ If dR = dL → Δθ = 0 (straight ahead)
✓ If dR > dL → Δθ > 0 → robot turns left (counterclockwise)

Step C — Deriving d_center (Distance Traveled)

Add equations [1] and [2]. Since d_center = R × Δθ:

dR + dL = Δθ × (R + L/2) + Δθ × (R − L/2) = 2 × R × Δθ = 2 × d_center

d_center = (dR + dL) / 2

The robot's center point travels the average of the two wheel distances.

Step D — Why θ + Δθ/2 and Not Just θ?

During the time step, the robot's heading sweeps continuously from θ at the start to θ + Δθ at the end. We need to choose a heading to use in the position update. Three options:

MethodHeading UsedError
Forward Eulerθ (start)O(Δθ) — large error when turning sharply
Backward Eulerθ + Δθ (end)O(Δθ) — same magnitude, opposite bias
Midpoint Rule ✓θ + Δθ/2 (middle)O(Δθ²) — much smaller; exact for constant curvature

The midpoint formula is not just more accurate — it gives the exact result for a circular arc, proven by integration:

Proof: Exact Arc Integral = Midpoint Formula x_exact = x + R × [sin(θ+Δθ) − sin(θ)]

Using sum-to-product identity: sin(A) − sin(B) = 2·cos((A+B)/2)·sin((A−B)/2)

= x + R × 2·cos(θ + Δθ/2)·sin(Δθ/2)
= x + (R·Δθ) × [sin(Δθ/2)/(Δθ/2)] × cos(θ + Δθ/2) ... but R·Δθ = d_center

For small Δθ: sin(Δθ/2)/(Δθ/2) → 1, giving:
= x + d_center × sin(θ + Δθ/2)   ✓ exact for circular arc

Step E — Complete Pose Update Equations

d_center = (dR + dL) / 2
Δθ      = (dR − dL) / L

x_new = x_old + d_center × sin(θ_old + Δθ/2)
y_new = y_old + d_center × cos(θ_old + Δθ/2)
θ_new = θ_old + Δθ

Self-check (θ = 0, straight ahead):
x_new = x + d·sin(0) = x + 0    ✓ x unchanged
y_new = y + d·cos(0) = y + d    ✓ y increases

Self-check (θ = 90°, facing +x, straight ahead):
x_new = x + d·sin(90°) = x + d    ✓ x increases
y_new = y + d·cos(90°) = y + 0    ✓ y unchanged

1.5 Odometry Drift and Why We Need More Sensors

Odometry is an integration process — errors accumulate at every step. In typical indoor environments, position error grows at roughly 1–5% of total distance. After 10 meters, that is 10–50 cm of error. Causes include:

Arduino C++
// ============================================================
// Differential Drive Odometry on Arduino
// Using quadrature encoders on both wheels
// Angle convention: theta measured from y-axis (vertical)
// theta=0 → robot faces +y → forward motion increases y
// ============================================================

// --- Pin Definitions ---
#define ENC_LEFT_A   2    // Must be interrupt pin
#define ENC_LEFT_B   4
#define ENC_RIGHT_A  3    // Must be interrupt pin
#define ENC_RIGHT_B  5

// --- Robot Parameters ---
const float WHEEL_DIAMETER  = 0.07;    // meters
const float WHEEL_BASE      = 0.20;    // meters
const int   TICKS_PER_REV   = 360;

// --- Encoder Counters (volatile = modified inside ISR) ---
volatile long ticksLeft  = 0;
volatile long ticksRight = 0;
long prevTicksLeft  = 0;
long prevTicksRight = 0;

// --- Robot Pose ---
float x     = 0.0;   // meters
float y     = 0.0;   // meters
float theta = 0.0;   // radians (from y-axis, ccw = positive)

// --- Interrupt Service Routines ---
// Direction: if B is HIGH when A rises, wheel turns forward (+1)
void ISR_Left()  { ticksLeft  += (digitalRead(ENC_LEFT_B)  == HIGH) ? 1 : -1; }
void ISR_Right() { ticksRight += (digitalRead(ENC_RIGHT_B) == HIGH) ? 1 : -1; }

void setup() {
  Serial.begin(115200);
  pinMode(ENC_LEFT_A,  INPUT_PULLUP);
  pinMode(ENC_LEFT_B,  INPUT_PULLUP);
  pinMode(ENC_RIGHT_A, INPUT_PULLUP);
  pinMode(ENC_RIGHT_B, INPUT_PULLUP);
  attachInterrupt(digitalPinToInterrupt(ENC_LEFT_A),  ISR_Left,  RISING);
  attachInterrupt(digitalPinToInterrupt(ENC_RIGHT_A), ISR_Right, RISING);
}

void loop() {
  delay(50);  // Update every 50ms

  // --- Compute tick deltas since last cycle ---
  long dTicksL = ticksLeft  - prevTicksLeft;
  long dTicksR = ticksRight - prevTicksRight;
  prevTicksLeft  = ticksLeft;
  prevTicksRight = ticksRight;

  // --- Convert ticks to distance ---
  float distPerTick = (PI * WHEEL_DIAMETER) / TICKS_PER_REV;
  float dL = dTicksL * distPerTick;
  float dR = dTicksR * distPerTick;

  // --- Compute center distance and heading change ---
  float d_center = (dR + dL) / 2.0;
  float dTheta   = (dR - dL) / WHEEL_BASE;

  // --- Update pose using midpoint rule (vertical-axis convention) ---
  float midTheta = theta + dTheta / 2.0;
  x     += d_center * sin(midTheta);   // sin for x (perpendicular to y-axis)
  y     += d_center * cos(midTheta);   // cos for y (along y-axis)
  theta += dTheta;

  // --- Wrap theta to [-PI, PI] ---
  while (theta >  PI) theta -= 2 * PI;
  while (theta < -PI) theta += 2 * PI;

  // --- Output pose ---
  Serial.print("x=");       Serial.print(x, 4);
  Serial.print(" m  y=");   Serial.print(y, 4);
  Serial.print(" m  deg="); Serial.println(theta * 180.0 / PI, 2);
}

    return ticks * (np.pi * WHEEL_DIAMETER) / TICKS_PER_REV

def update_pose(x, y, theta, ticks_left, ticks_right):
    """
    Update robot pose from encoder ticks.
    Convention: theta measured from y-axis (vertical).
    theta=0 means robot faces +y direction.
    """
    dL = ticks_to_distance(ticks_left)
    dR = ticks_to_distance(ticks_right)

    d_theta  = (dR - dL) / WHEEL_BASE

    mid_theta = theta + d_theta / 2.0

    x_new     = x + d_center * np.sin(mid_theta)
    y_new     = y + d_center * np.cos(mid_theta)
    theta_new = theta + d_theta

    # Normalize theta to [-pi, pi]
    theta_new = (theta_new + np.pi) % (2 * np.pi) - np.pi

    return x_new, y_new, theta_new

    """
    Simulate executing a list of (ticks_left, ticks_right) commands.
    Adds Gaussian noise to simulate real-world encoder imperfections.
    Returns two paths: ideal (no noise) and noisy (with slip).
    """

    noisy_path = [(0.0, 0.0)]

    nx, ny, nt = 0.0, 0.0, 0.0

        nx, ny, nt = update_pose(nx, ny, nt, tL + noise_L, tR + noise_R)
        noisy_path.append((nx, ny))

    return ideal_path, noisy_path

2.1 What Is Ackermann Steering?

Ackermann steering is the geometry used in all cars, trucks, and many wheeled robots. Instead of spinning wheels at different speeds to turn, the robot physically pivots the front wheels using a steering mechanism. The critical design principle:

🔑 The Ackermann Principle
When a four-wheeled vehicle turns, the inner and outer front wheels trace circles of different radii about the same center point. The inner wheel's circle is smaller, so it must turn to a larger angle than the outer wheel. If both wheels turn to the same angle, the tyres scrub sideways — causing wear, loss of traction, and inaccurate odometry.
Turn Center (ICC) * /| / | [FL]↙ / | [FR]↗ inner / | outer / | =====[ CHASSIS ]===== | | [RL] [RR] ← rear wheels (driven) δ_L (inner) > δ_R (outer) — this is the Ackermann condition

Building the Front Steering Mechanism Step by Step

1

Design and Make the Steering Knuckles

Each knuckle needs three features: (1) a horizontal wheel axle hole, (2) a vertical kingpin pivot hole, and (3) a tie-rod arm extending inward at the Ackermann correction angle. Calculate this angle: arctan(T / 2L). For T=0.16 m, L=0.20 m → arctan(0.4) ≈ 21.8°. The tie-rod arm must point inward by this angle to ensure the inner wheel turns more than the outer. 3D-print from PLA or cut from 3 mm aluminum plate.

2

Install the Kingpins

Each kingpin is an M4 bolt that passes vertically through the chassis and through the top of the steering knuckle. The knuckle pivots freely around this bolt. Fit a thin nylon washer between the chassis underside and the knuckle top for smooth rotation. Secure with two nuts: inner nut for tension adjustment, outer nut as lock nut. Goal: zero wobble, but smooth free pivoting.

3

Attach the Tie Rod

Cut the aluminum rod to ~120 mm (for 160 mm track width). Thread M3 clevises onto both ends — lock with nuts. Connect the clevises to the holes on each knuckle's tie-rod arm. Test manually: pushing the rod left should turn both front wheels left, with the inner wheel turning to a larger angle than the outer. If both wheels turn to the same angle, re-check the Ackermann angle on your knuckles.

4

Connect the Servo via Pushrod

Mount the servo centered between the two front wheels on the chassis. Cut a 30–50 mm rigid pushrod (thick wire or small bolt). Attach one end to the servo horn and the other to a point on the tie rod or directly to one knuckle arm. When the servo rotates, the pushrod moves the tie rod, steering both front wheels simultaneously.

5

Add Potentiometer Feedback

Mount a 10 kΩ potentiometer so its shaft is co-axial with one of the kingpin bolts. As the wheel steers left/right, the pot wiper rotates, giving an analog voltage proportional to the steering angle. Connect to Arduino A0. Calibrate: record the ADC value (0–1023) at full left, center, and full right. This gives you accurate real-time steering angle measurements independent of the servo command.

6

Build the Rear Drive

Mount one gear motor per rear wheel using L-brackets or 3D-printed mounts. For simplest construction, use two independent motors (both commanded to the same speed for straight driving). Mount one encoder disc on each motor shaft (before the gearbox for higher resolution). Route all cables along the chassis using zip ties, ending at the Arduino Mega's interrupt and analog pins.

2.3 The Ackermann Condition — Full Derivation

For all wheels to roll without scrubbing, they must all rotate about the same instantaneous center. Let T = track width, L = wheelbase:

Inner front wheel radius from ICC: R_inner = R − T/2
Outer front wheel radius from ICC: R_outer = R + T/2

cot(δ_L) = R_inner / L = (R − T/2) / L
cot(δ_R) = R_outer / L = (R + T/2) / L

Subtracting:
cot(δ_R) − cot(δ_L) = T / L

This is the Ackermann condition. Your steering geometry satisfies it if and only if
the outer wheel turns less than the inner wheel by exactly T/L in cotangent space.

2.4 Ackermann Kinematic Model — Full Derivation

We use the bicycle model: four wheels simplified to one front steered wheel and one rear driven wheel, both centered on the robot's centreline.

Turning radius from rear axle center R = L / tan(δ)

Angular velocity of robot body ω = v / R = v × tan(δ) / L

Over time step dt d  = v × dt              (distance traveled by rear wheel)
dθ = (v/L) × tan(δ) × dt   (heading change)

Pose update (midpoint rule, vertical-axis convention)
d  = ticks × (π × diameter) / (TPR)
dθ = (v / L) × tan(δ) × dt

x_new = x + d × sin(θ + dθ/2)
y_new = y + d × cos(θ + dθ/2)
θ_new = θ + dθ

2.6 Holonomic vs. Non-Holonomic Robots

A robot is holonomic if it can move in any direction at any instant, regardless of its current heading. In 2D space, a robot has 3 degrees of freedom: x, y, and θ. A holonomic robot can change any of them independently.

⚙ Non-Holonomic Constraint (Differential Drive and Ackermann)
ẋ·sin(θ) − ẏ·cos(θ) = 0    (no sideways velocity — the no-slip constraint)

This means x and y cannot change independently of θ. To move sideways, you must first rotate. Like parallel parking a car — you cannot simply slide sideways.

const int POT_FULL_LEFT  = 200;   // ADC value at max left steer
const int POT_CENTER     = 512;   // ADC value with wheels straight
const int POT_FULL_RIGHT = 820;   // ADC value at max right steer

// --- Encoder State ---
volatile long rearTicks = 0;
long prevRearTicks = 0;

// --- Pose ---
float x = 0.0, y = 0.0, theta = 0.0;

unsigned long lastTimeMs = 0;
Servo steeringServo;

// ISR for rear encoder
void ISR_Rear() {
  rearTicks += (digitalRead(ENC_REAR_B) == HIGH) ? 1 : -1;
}

float readSteeringAngleDeg() {
  int raw = analogRead(STEER_POT);
  float deg;
  if (raw <= POT_CENTER) {
    // Left of center → negative angle
    deg = -MAX_STEER_DEG * (float)(POT_CENTER - raw) / (POT_CENTER - POT_FULL_LEFT);
  } else {
    // Right of center → positive angle
    deg = MAX_STEER_DEG * (float)(raw - POT_CENTER) / (POT_FULL_RIGHT - POT_CENTER);
  }
  // Clamp to safe range
  return constrain(deg, -MAX_STEER_DEG, MAX_STEER_DEG);
}

void setup() {
  Serial.begin(115200);
  Serial.println("Ackermann Odometry — Robotics & Intelligence Nigeria Ring");

  pinMode(ENC_REAR_A, INPUT_PULLUP);
  pinMode(ENC_REAR_B, INPUT_PULLUP);
  attachInterrupt(digitalPinToInterrupt(ENC_REAR_A), ISR_Rear, RISING);

  steeringServo.attach(STEER_SERVO);
  steeringServo.write(90);  // Center position = 90° on servo

  lastTimeMs = millis();
  Serial.println("Ready. Waiting for motion...");
}

void loop() {
  unsigned long now = millis();
  float dt = (now - lastTimeMs) / 1000.0;   // seconds
  if (dt < 0.05) return;   // Update at 20 Hz max
  lastTimeMs = now;

  // --- Read encoder ticks since last cycle ---
  long dTicks = rearTicks - prevRearTicks;
  prevRearTicks = rearTicks;

  // --- Read steering angle from potentiometer ---
  float steer_deg = readSteeringAngleDeg();
  float steer_rad = steer_deg * PI / 180.0;

  // --- Compute distance and velocity ---
  float d = dTicks * (PI * WHEEL_DIAMETER) / TICKS_PER_REV;
  float v = d / dt;   // m/s

  // --- Compute heading change ---
  float dTheta = (v / WHEELBASE) * tan(steer_rad) * dt;
  // Alternative exact form: dTheta = d * tan(steer_rad) / WHEELBASE

  // --- Update pose (midpoint rule, vertical-axis convention) ---
  float midTheta = theta + dTheta / 2.0;
  x     += d * sin(midTheta);
  y     += d * cos(midTheta);
  theta += dTheta;

  // Normalize theta to [-PI, PI]
  while (theta >  PI) theta -= 2.0 * PI;
  while (theta < -PI) theta += 2.0 * PI;

  // --- Print pose ---
  if (abs(dTicks) > 0 || abs(steer_deg) > 1.0) {
    Serial.print("x=");      Serial.print(x, 4);
    Serial.print(" y=");     Serial.print(y, 4);

    Serial.print("° steer="); Serial.print(steer_deg, 1);
    Serial.print("° v=");    Serial.print(v, 3);
    Serial.println("m/s");
  }
}

3.1 Physics of LiDAR

LiDAR — Light Detection and Ranging — fires a laser pulse, times how long it takes to bounce back off an obstacle, and computes distance from the speed of light:

distance = (c × t_flight) / 2     c = 3 × 10⁸ m/s

Example: t_flight = 10 ns = 10 × 10⁻⁹ s
distance = (3×10⁸ × 10×10⁻⁹) / 2 = 3/2 = 1.5 meters

Division by 2 because the pulse must travel to the obstacle AND back.

A 2D spinning LiDAR fires continuously while rotating a mirror through 360°, producing one (angle, distance) pair per degree — 360 readings per scan, once every ~100–200 ms.

3.4 Occupancy Grid Maps

An occupancy grid divides the environment into a 2D matrix of cells (typically 5 cm × 5 cm). Each cell stores the probability that it is occupied by an obstacle. We use log-odds for numerically stable additive updates:

P(occupied) → stored as log-odds L = log(P / (1−P))

Update occupied cell: L ← L + L_OCC    (e.g., +0.9)
Update free cell: L ← L + L_FREE   (e.g., −0.4)

Convert back: P = e^L / (1 + e^L) = sigmoid(L)

Cells start at L=0 (P=0.5, unknown).
A cell hit repeatedly: L → +5 (P → 0.993, very likely occupied)
A beam passes through repeatedly: L → −5 (P → 0.007, very likely free)

Bresenham's Line Algorithm determines which cells a LiDAR beam passes through on its way to the endpoint. All traversed cells are marked free; only the endpoint cell is marked occupied.

3.5 ICP — Iterative Closest Point

ICP finds the rotation R and translation t that best aligns two point clouds. It is the core of scan matching in LiDAR SLAM — used to estimate how the robot moved between consecutive scans.

1

Initialize

Use the odometry estimate as the initial transformation guess. Apply it to the source scan before starting iterations.

2

Find Nearest Correspondences

For every point in the source scan, find the closest point in the target scan using a KD-tree (O(log N) per query — much faster than brute-force O(N)).

3

Compute Optimal Transform (SVD)

Center both point sets at their centroids. Compute covariance H = Σ(s_i' × t_j'ᵀ). SVD decompose: H = UΣVᵀ. Rotation: R = V×Uᵀ. Translation: t = t̄ − R×s̄. This gives the transformation that minimises Σ||R·sᵢ + t − tⱼ||².

4

Apply and Accumulate

Apply R and t to the source scan. Accumulate the total transformation across iterations.

5

Check Convergence

Compute MSE = mean of squared distances between corresponding points. If MSE < ε (typically 0.001 m²) or improvement is below tolerance — stop. Otherwise go to Step 2.

# Occupancy Grid Map Builder with Log-Odds Updates
# Uses RPLiDAR scans + robot pose (from odometry or EKF)
# Robotics and Intelligence Nigeria Ring
# ============================================================

import numpy as np
import matplotlib.pyplot as plt
import matplotlib.animation as animation
from rplidar import RPLidar

# --- Grid configuration ---
MAP_SIZE_M  = 10.0    # Physical size: 10m × 10m
RESOLUTION  = 0.05    # Each cell = 5cm × 5cm
GRID_CELLS  = int(MAP_SIZE_M / RESOLUTION)   # = 200 × 200 cells
ORIGIN      = GRID_CELLS // 2               # Robot starts at grid center

# --- Log-odds update values ---
L_OCC  =  0.9    # Added to occupied cell log-odds
L_FREE = -0.4    # Added to free cell log-odds
L_MIN  = -5.0    # Minimum log-odds (very confident: free)
L_MAX  =  5.0    # Maximum log-odds (very confident: occupied)

# --- Initialize grid: all cells unknown (L=0 → P=0.5) ---
log_odds_grid = np.zeros((GRID_CELLS, GRID_CELLS), dtype=np.float32)


def world_to_grid(wx, wy):
    """Convert world coordinates (meters) to grid cell indices."""
    gx = int(wx / RESOLUTION) + ORIGIN
    gy = int(wy / RESOLUTION) + ORIGIN
    return (np.clip(gx, 0, GRID_CELLS-1),
            np.clip(gy, 0, GRID_CELLS-1))


def bresenham(x0, y0, x1, y1):
    """
    Bresenham's line algorithm.
    Returns list of (x, y) grid cell indices along the line from (x0,y0) to (x1,y1).
    Used to find which cells a LiDAR beam passes through.
    """
    cells = []
    dx, dy = abs(x1-x0), abs(y1-y0)
    sx = 1 if x0 < x1 else -1
    sy = 1 if y0 < y1 else -1
    err = dx - dy

    while True:
        cells.append((x0, y0))
        if x0 == x1 and y0 == y1:
            break
        e2 = 2 * err
        if e2 > -dy:
            err -= dy
            x0  += sx
        if e2 < dx:
            err += dx
            y0  += sy
    return cells


def update_map(robot_x, robot_y, robot_theta, scan_angles, scan_dists):
    """
    Update the occupancy grid given robot pose and a LiDAR scan.

    Parameters:
        robot_x, robot_y:  Robot position in world coordinates (meters)
        robot_theta:       Robot heading from y-axis (vertical-axis convention)
        scan_angles:       Array of scan angles in radians (LiDAR frame)
        scan_dists:        Array of scan distances in meters
    """
    global log_odds_grid

    rx_g, ry_g = world_to_grid(robot_x, robot_y)

    for angle, dist in zip(scan_angles, scan_dists):
        if dist < 0.15 or dist > 8.0:
            continue   # Ignore out-of-range readings

        # Convert LiDAR angle to world frame (add robot heading)
        # angle=0 in LiDAR frame = robot's forward direction
        world_angle = angle + robot_theta

        # Endpoint of LiDAR beam in world coordinates
        # Vertical-axis convention: sin for x, cos for y
        end_x = robot_x + dist * np.sin(world_angle)
        end_y = robot_y + dist * np.cos(world_angle)
        ex_g, ey_g = world_to_grid(end_x, end_y)

        # All cells along the beam (except endpoint) are FREE
        free_cells = bresenham(rx_g, ry_g, ex_g, ey_g)
        for (cx, cy) in free_cells[:-1]:
            if 0 <= cx < GRID_CELLS and 0 <= cy < GRID_CELLS:
                log_odds_grid[cy, cx] = max(L_MIN,
                                            log_odds_grid[cy, cx] + L_FREE)

        # Endpoint cell is OCCUPIED
        if 0 <= ex_g < GRID_CELLS and 0 <= ey_g < GRID_CELLS:
            log_odds_grid[ey_g, ex_g] = min(L_MAX,
                                             log_odds_grid[ey_g, ex_g] + L_OCC)


def get_probability_map():
    """Convert log-odds grid to probability map [0, 1] using sigmoid."""
    return 1.0 / (1.0 + np.exp(-log_odds_grid))

import numpy as np
from scipy.spatial import KDTree
import matplotlib.pyplot as plt


def apply_transform_2d(points, dx, dy, dtheta):
    """Apply 2D rigid transformation to Nx2 point array."""
    c, s = np.cos(dtheta), np.sin(dtheta)
    R = np.array([[c, -s], [s, c]])
    return (R @ points.T).T + np.array([dx, dy])


def icp(source, target, max_iterations=50, tolerance=1e-4, max_dist=0.5):
    """
    Iterative Closest Point scan matching.

    Parameters:
        source:         Nx2 array of (x,y) points — current LiDAR scan
        target:         Mx2 array of (x,y) points — reference / map scan
        max_iterations: Maximum ICP iterations before stopping
        tolerance:      Stop when MSE improvement per iteration < tolerance
        max_dist:       Reject correspondence pairs farther apart than this (m)

    Returns:
        total_dx, total_dy, total_theta: Best-fit transformation
        mse_history:  List of MSE values per iteration (for convergence plot)
    """
    total_dx, total_dy, total_theta = 0.0, 0.0, 0.0
    current = source.copy()
    mse_history = []
    prev_mse = np.inf

    # Build KD-tree once on the target for fast nearest-neighbour search
    tree = KDTree(target)

    for iteration in range(max_iterations):

        # STEP 2: Find nearest-neighbour correspondences
        distances, indices = tree.query(current)
        valid = distances < max_dist

        if valid.sum() < 10:
            print(f"WARNING: Only {valid.sum()} valid pairs. Stopping early.")
            break

        src_pts = current[valid]
        tgt_pts = target[indices[valid]]

        # STEP 3a: Compute centroids
        sc = src_pts.mean(axis=0)
        tc = tgt_pts.mean(axis=0)

        # STEP 3b: Centre both point sets
        src_c = src_pts - sc
        tgt_c = tgt_pts - tc

        # STEP 3c: Covariance matrix H (2×2)
        H = src_c.T @ tgt_c

        # STEP 3d: SVD — H = U × Σ × Vᵀ
        U, S, Vt = np.linalg.svd(H)

        # STEP 3e: Rotation matrix R
        # Handle reflection (det must be +1, not -1)
        d = np.linalg.det(Vt.T @ U.T)
        D = np.diag([1.0, 1.0 if d > 0 else -1.0])
        R = Vt.T @ D @ U.T

        # Extract rotation angle from 2D rotation matrix
        dtheta = np.arctan2(R[1, 0], R[0, 0])

        # STEP 3f: Translation
        t = tc - R @ sc
        dx, dy = t[0], t[1]

        # STEP 4: Apply transformation to current source
        current = apply_transform_2d(current, dx, dy, dtheta)

        # Accumulate total transformation (accounting for already-applied rotation)
        c_acc, s_acc = np.cos(total_theta), np.sin(total_theta)
        total_dx    = c_acc * dx - s_acc * dy + total_dx
        total_dy    = s_acc * dx + c_acc * dy + total_dy
        total_theta += dtheta

        # STEP 5: Check convergence
        mse = float(np.mean(distances[valid] ** 2))
        mse_history.append(mse)

        if abs(prev_mse - mse) < tolerance:
            print(f"ICP converged at iteration {iteration+1}  MSE={mse:.6f} m²")
            break
        prev_mse = mse

    return total_dx, total_dy, total_theta, mse_history

4.1 Why Not GPS Indoors?

GPS works by receiving time-stamped signals from 4+ satellites. Indoors, buildings absorb and reflect these signals — accuracy drops to 10–50 m or fails entirely. Indoor Positioning Systems (IPS) use signals that work reliably inside buildings.

4.2 UWB — Ultra-Wideband Positioning

UWB sends extremely short radio pulses (nanosecond-wide) across a wide frequency band (3.1–10.6 GHz). The key measurement is Time of Flight (ToF). Because the pulses are so short, even nanosecond timing differences give centimetre-level accuracy.

Two-Way Ranging (TWR) eliminates the need for synchronised clocks:

Anchor A ──[T1: send]──→ Tag B
Anchor A ←──[T3: reply]── Tag B (T2 = receive time, T3 = reply send time)
Anchor A receives reply at T4

t_flight = [(T4 − T1) − (T3 − T2)] / 2
distance = c × t_flight = 3×10⁸ m/s × t_flight

Example: t_flight = 5 ns → distance = 1.5 m

Common UWB hardware for labs: Decawave DWM1000, Decawave DWM1001, Pozyx positioning kit (complete kit with Arduino library), MDEK1001 development kit.

4.3 Trilateration — Full Derivation

With 3 anchors at known positions and measured distances, the robot is at the intersection of 3 circles:

(x−x₁)² + (y−y₁)² = d₁²   ...[1]
(x−x₂)² + (y−y₂)² = d₂²   ...[2]
(x−x₃)² + (y−y₃)² = d₃²   ...[3]

Linearize: subtract [1] from [2] Expand both equations, subtract:
−2x·x₂ + x₂² − (−2x·x₁ + x₁²) − 2y·y₂ + y₂² + 2y·y₁ − y₁² = d₁² − d₂²
→ 2(x₂−x₁)·x + 2(y₂−y₁)·y = d₁²−d₂² + x₂²−x₁² + y₂²−y₁²

Similarly subtract [1] from [3] 2(x₃−x₁)·x + 2(y₃−y₁)·y = d₁²−d₃² + x₃²−x₁² + y₃²−y₁²

Linear system Ax = b
A = |2(x₂−x₁) 2(y₂−y₁)|    b = |d₁²−d₂²+x₂²−x₁²+y₂²−y₁²|
    |2(x₃−x₁) 2(y₃−y₁)|        |d₁²−d₃²+x₃²−x₁²+y₃²−y₁²|

3 anchors: x = A⁻¹·b    (exact solution)
N anchors: x = (AᵀA)⁻¹·Aᵀ·b   (least squares)

import numpy as np
import matplotlib.pyplot as plt
import matplotlib.patches as patches


def trilaterate(anchors, distances):
    """
    Compute 2D robot position from N anchors and distances.
    Uses least-squares for N >= 3 anchors (minimum: 3).

    Parameters:
        anchors:   Nx2 array [[x1,y1], [x2,y2], ...] — anchor positions (m)
        distances: N-element array [d1, d2, ...] — measured distances (m)

    Returns:
        position:  [x, y] estimated robot position (m)
        residuals: fitting error vector
    """
    anchors   = np.array(anchors,   dtype=float)
    distances = np.array(distances, dtype=float)
    n = len(anchors)

    if n < 3:
        raise ValueError("Need at least 3 anchors for 2D positioning")

    # Use first anchor as reference
    x1, y1 = anchors[0];  d1 = distances[0]

    A_rows, b_rows = [], []
    for i in range(1, n):
        xi, yi = anchors[i];  di = distances[i]
        A_rows.append([2*(xi-x1), 2*(yi-y1)])
        b_rows.append(d1**2 - di**2 + xi**2 - x1**2 + yi**2 - y1**2)

    A = np.array(A_rows)   # (n-1) × 2
    b = np.array(b_rows)   # (n-1,)

    # Least squares: minimise ||Ax - b||²
    pos, residuals, _, _ = np.linalg.lstsq(A, b, rcond=None)
    return pos, residuals


def simulate_uwb(true_pos, anchors, noise_std=0.05):
    """Simulate UWB range measurements with Gaussian noise."""
    true_pos = np.array(true_pos)
    dists = []
    for a in np.array(anchors):
        d_true = np.linalg.norm(true_pos - a)
        dists.append(max(0.05, d_true + np.random.normal(0, noise_std)))
    return dists


def run_position_tracking(true_path, anchors, noise_std=0.07, n_trials=1):
    """
    Simulate robot moving along a path, computing UWB position at each step.
    Returns estimated path and errors.
    """
    est_path = []
    errors   = []

    for true_pos in true_path:
        # Average over multiple measurements to reduce noise
        estimates = []
        for _ in range(n_trials):
            dists = simulate_uwb(true_pos, anchors, noise_std)
            pos, _ = trilaterate(anchors, dists)
            estimates.append(pos)

        est_pos = np.mean(estimates, axis=0)
        est_path.append(est_pos)
        err = np.linalg.norm(np.array(true_pos) - est_pos)
        errors.append(err)

    return np.array(est_path), np.array(errors)

class WiFiFingerprinter:
    """
    Wi-Fi fingerprint indoor positioning system.

    Workflow:
    1. Call collect_fingerprint(label) at each known location during survey
    2. Call locate() at runtime to estimate current position
    """

    def __init__(self, db_file='wifi_fingerprint_db.json'):
        self.db_file  = db_file
        self.database = {}   # {location: {bssid: avg_rssi_dbm}}

        if WIFI_AVAILABLE:
            self.wifi      = pywifi.PyWiFi()
            self.iface     = self.wifi.interfaces()[0]
        self.load_database()

    # ── Scanning ─────────────────────────────────────────────
    def scan_wifi(self, num_samples=5):
        """
        Perform Wi-Fi scan and return averaged RSSI for each BSSID.
        Multiple samples reduce noise from signal fluctuations.
        Returns: {bssid_string: avg_rssi_dbm}
        """
        if not WIFI_AVAILABLE:
            return self._simulate_scan()

        accumulated = {}
        counts      = {}

        for _ in range(num_samples):
            self.iface.scan()
            time.sleep(0.5)   # Allow scan to complete
            results = self.iface.scan_results()

            for ap in results:
                bssid = ap.bssid
                rssi  = ap.signal   # dBm, typically -30 to -90
                accumulated[bssid] = accumulated.get(bssid, 0) + rssi
                counts[bssid]      = counts.get(bssid, 0) + 1

        # Return averaged values
        return {b: accumulated[b] / counts[b] for b in accumulated}

    def _simulate_scan(self):
        """Simulation fallback — returns fake RSSI values with noise."""
        return {
            'AA:BB:CC:DD:EE:01': -45 + np.random.normal(0, 3),
            'AA:BB:CC:DD:EE:02': -62 + np.random.normal(0, 3),
            'AA:BB:CC:DD:EE:03': -78 + np.random.normal(0, 3),
            'AA:BB:CC:DD:EE:04': -88 + np.random.normal(0, 4),
            'AA:BB:CC:DD:EE:05': -70 + np.random.normal(0, 5),
        }

    # ── Offline Phase: Survey ─────────────────────────────────
    def collect_fingerprint(self, location_label, num_samples=10):
        """
        Collect and store Wi-Fi fingerprint at a known location.
        Stand still while calling this — movement during scan adds error.

        Parameters:
            location_label: descriptive name, e.g. 'Lab Entrance'
            num_samples:    number of scan repetitions to average
        """
        print(f"\nCollecting fingerprint for: '{location_label}'")
        print(f"Stay completely still. Taking {num_samples} samples...")

        scan_data = self.scan_wifi(num_samples=num_samples)

        if location_label not in self.database:
            self.database[location_label] = {}

        # Running average: new measurement blended with stored value
        for bssid, rssi in scan_data.items():
            if bssid in self.database[location_label]:
                old = self.database[location_label][bssid]
                self.database[location_label][bssid] = (old + rssi) / 2.0
            else:
                self.database[location_label][bssid] = rssi

        self.save_database()
        print(f"Saved: {len(scan_data)} APs recorded for '{location_label}'")
        return scan_data

    # ── Online Phase: Locate ──────────────────────────────────
    def locate(self, k_neighbors=3):
        """
        Estimate current location by KNN matching in RSSI space.

        Parameters:
            k_neighbors: number of top candidates to report

        Returns:
            best_location: string label of best-match location
            confidence:    score 0–1 (higher = more confident)
            all_results:   sorted list of (location, distance, confidence)
        """
        if not self.database:
            raise RuntimeError("Database is empty! Run collect_fingerprint() first.")

        current_scan = self.scan_wifi()

        # Build a consistent feature vector using ALL known BSSIDs
        all_bssids = sorted(set(
            bssid for loc in self.database.values() for bssid in loc
        ))
        NO_SIGNAL = -100.0   # Value for unseen APs (very weak)

        def make_vector(scan_dict):
            return np.array([scan_dict.get(b, NO_SIGNAL) for b in all_bssids])

        current_vec = make_vector(current_scan)

        # Compute Euclidean distance in RSSI space to each stored fingerprint
        results = []

5.1 The Problem — Noisy Sensors, Imperfect Motion

Every sensor lies — randomly. Every motion model has imperfections. Given multiple noisy information sources, the Kalman Filter gives the single mathematically optimal answer: what is the best possible estimate of the robot's true state?

🏆 Why Kalman is Optimal
For systems with Gaussian noise, it is mathematically proven that no other linear estimator can achieve a lower mean-square error. The Kalman Filter was invented by Rudolf Kálmán in 1960 and guided the Apollo lunar landing.

5.2 Probability Prerequisite

Sensor noise follows a Gaussian (Normal) distribution. A sensor with noise σ = 0.08 m means ~68% of readings fall within ±0.08 m of the true value, and ~95% fall within ±0.16 m.

When tracking both x and y simultaneously, we need a covariance matrix P:

P = | σ_xx σ_xy |
    | σ_yx σ_yy |

σ_xx = variance of x (uncertainty squared)
σ_yy = variance of y
σ_xy = how much x and y errors are correlated (usually 0 if independent)

5.3 The Kalman Filter System Model

State Transition Model (motion model) x_k = A × x_{k-1} + B × u_k + w_k
w_k ~ N(0, Q) — process noise

Measurement Model z_k = H × x_k + v_k
v_k ~ N(0, R) — measurement noise

A = state transition matrix (how state evolves without input)
B = control input matrix    u_k = control input (motor command)
H = observation matrix (which part of state does sensor measure?)
Q = process noise covariance    R = measurement noise covariance

Example for 1D robot (position + velocity):

A = |1 dt|   x_k = |pos_k| = |pos + vel×dt|
    |0 1|        |vel_k| |vel |

H = [1 0]   (UWB measures position only, not velocity)

5.4 PREDICT Step

Predicted state:
x̂_{k|k-1} = A × x̂_{k-1|k-1} + B × u_k

Predicted covariance:
P_{k|k-1} = A × P_{k-1|k-1} × Aᵀ + Q

After predict: P grows — process noise Q is added.
We become less certain the further we predict ahead without a measurement.

5.5 UPDATE Step

Innovation (how far off is the measurement from prediction?):
ỹ_k = z_k − H × x̂_{k|k-1}

Innovation covariance:
S_k = H × P_{k|k-1} × Hᵀ + R

Kalman Gain:
K_k = P_{k|k-1} × Hᵀ × S_k⁻¹

Updated state:
x̂_{k|k} = x̂_{k|k-1} + K_k × ỹ_k

Updated covariance:
P_{k|k} = (I − K_k × H) × P_{k|k-1}

5.6 Kalman Gain Intuition

💡 Understanding K
K = 0 → ignore the measurement entirely. Trust prediction only.
K = 1 → trust the measurement completely. Ignore prediction.
K = 0.5 → equal weight to both.

K is computed automatically as: K = P / (P + R)
Large P (uncertain prediction) + small R (accurate sensor) → K near 1 → trust sensor
Small P (confident prediction) + large R (noisy sensor) → K near 0 → trust prediction

After the update, P always decreases — incorporating a measurement always reduces uncertainty.

dt = 0.1 s  |  Q = diag([0.01, 0.1])  |  R = [0.25] (UWB, σ = 0.5 m)
H = [1, 0] — measures position only
Initial: x̂₀ = [0, 0], P₀ = diag([1.0, 0.1])

PREDICT (acceleration command u = 0.5 m/s²):
x̂_{1|0} = A×[0,0]ᵀ + B×0.5 = [0.0025, 0.05]ᵀ
P_{1|0}[0,0] ≈ 1.011 m²   (grew slightly due to Q)

UWB measurement: z = 0.08 m

UPDATE:
Innovation: ỹ = 0.08 − [1,0]×[0.0025,0.05]ᵀ = 0.08 − 0.0025 = 0.0775

UWB measurement: z = 0.15 m

K₀ = 0.210 / (0.210 + 0.25) = 0.456 (lower than Cycle 1 — more confident now)
x̂_{2|2}[0] = 0.0722 + 0.456 × (0.15 − 0.0722) = 0.108 m
P_{2|2}[0,0] = (1 − 0.456) × 0.210 = 0.114 m² (was 0.210 → −46%)

After just 2 cycles, uncertainty dropped from 1.0 m² to 0.114 m² — 89% reduction!

5.8 Extended Kalman Filter (EKF)

The standard KF requires linear models. Robot kinematics involve sin and cos — which are nonlinear. The EKF linearises the nonlinear functions at each timestep using Jacobian matrices (the matrix of all first-order partial derivatives).

For our differential drive robot, state x = [x, y, θ]ᵀ:

Nonlinear motion function f(x, u) f = | x + d × sin(θ + Δθ/2) |
    | y + d × cos(θ + Δθ/2) |
    | θ + Δθ |

Jacobian F = ∂f/∂x (replaces matrix A in the EKF predict step) F = | 1 0 d × cos(θ + Δθ/2) |
    | 0 1 −d × sin(θ + Δθ/2) |
    | 0 0 1 |

EKF Predict: use F instead of A P_{k|k-1} = F × P × Fᵀ + Q    (same structure, different matrix)

import numpy as np
import matplotlib.pyplot as plt


class KalmanFilter2D:
    """
    Kalman Filter for 2D robot tracking.
    State: [x, y, vx, vy] — position and velocity in 2D.
    Vertical-axis convention throughout.
    """

    def __init__(self, dt=0.1, process_noise=0.01, meas_noise=0.1):
        self.dt = dt
        self.n  = 4   # State dimension: [x, y, vx, vy]

        # State transition matrix A
        # x_new = x + vx*dt,  y_new = y + vy*dt,  v unchanged
        self.A = np.array([
            [1, 0, dt,  0],
            [0, 1,  0, dt],
            [0, 0,  1,  0],
            [0, 0,  0,  1]
        ])

        # Observation matrix H — UWB measures x and y only
        self.H = np.array([
            [1, 0, 0, 0],
            [0, 1, 0, 0]
        ])

        # Process noise Q — how much we trust the motion model
        q = process_noise
        self.Q = np.diag([q, q, q*10, q*10])

        # Measurement noise R — how noisy the UWB sensor is
        r = meas_noise
        self.R = np.diag([r, r])

        # Initial state and covariance
        self.x = np.zeros((self.n, 1))
        self.P = np.eye(self.n) * 1.0   # Start with high uncertainty

    def predict_from_odometry(self, odometry_dx, odometry_dy):
        """
        Predict step driven by wheel odometry displacement.
        odometry_dx, odometry_dy: estimated displacement in meters this step.
        """
        # Update state: position += odometry, velocity = odometry/dt
        self.x[0] += odometry_dx
        self.x[1] += odometry_dy
        self.x[2]  = odometry_dx / self.dt
        self.x[3]  = odometry_dy / self.dt

        # Covariance grows with process noise
        self.P = self.A @ self.P @ self.A.T + self.Q

    def update_from_uwb(self, uwb_x, uwb_y):
        """
        Update step: incorporate UWB position measurement.
        Returns the Kalman gain (useful for analysis).
        """
        z = np.array([[uwb_x], [uwb_y]])

        # Innovation: difference between measurement and prediction
        innovation = z - self.H @ self.x

        # Innovation covariance
        S = self.H @ self.P @ self.H.T + self.R

        # Kalman Gain
        K = self.P @ self.H.T @ np.linalg.inv(S)

        # Update state and covariance (Joseph form for numerical stability)
        self.x = self.x + K @ innovation
        I_KH   = np.eye(self.n) - K @ self.H
        self.P = I_KH @ self.P @ I_KH.T + K @ self.R @ K.T

        return K

    def get_position(self):
        return float(self.x[0]), float(self.x[1])

    def get_std(self):
        return float(np.sqrt(self.P[0,0])), float(np.sqrt(self.P[1,1]))


def simulate_fusion(n_steps=100, uwb_every_n=5):
    """
    Simulate robot on a curved path, fusing odometry + UWB.
    Compares odometry-only vs Kalman-filtered estimates.
    """
    np.random.seed(42)
    dt   = 0.1
    kf   = KalmanFilter2D(dt=dt, process_noise=0.005, meas_noise=0.04)

    true_x, true_y           = [0.0], [0.0]
    odom_x, odom_y           = [0.0], [0.0]
    kf_x_list, kf_y_list     = [0.0], [0.0]
    uwb_x_list, uwb_y_list   = [], []
    kf_std_x                 = [1.0]

    ox, oy = 0.0, 0.0   # Odometry accumulated position

    for t in range(n_steps):
        t_sec = t * dt

        # True path: sinusoidal curve
        tx = 0.4 * np.sin(t_sec * 0.6)
        ty = 0.12 * t_sec ** 1.1

        prev_tx = true_x[-1] if len(true_x) > 1 else 0.0
        prev_ty = true_y[-1] if len(true_y) > 1 else 0.0

        dx_true = tx - prev_tx
        dy_true = ty - prev_ty

        true_x.append(tx)
        true_y.append(ty)

        # Odometry: true motion + wheel slip noise
        dx_odom = dx_true + np.random.normal(0, 0.008)
        dy_odom = dy_true + np.random.normal(0, 0.006)
        ox += dx_odom
        oy += dy_odom
        odom_x.append(ox)
        odom_y.append(oy)

        # KF Predict from odometry
        kf.predict_from_odometry(dx_odom, dy_odom)

        # KF Update from UWB every uwb_every_n steps
        if t % uwb_every_n == 0:
            uwb_meas_x = tx + np.random.normal(0, 0.07)
            uwb_meas_y = ty + np.random.normal(0, 0.07)
            uwb_x_list.append(uwb_meas_x)
            uwb_y_list.append(uwb_meas_y)
            kf.update_from_uwb(uwb_meas_x, uwb_meas_y)

        kfx, kfy = kf.get_position()
        sx, _    = kf.get_std()
        kf_x_list.append(kfx)
        kf_y_list.append(kfy)
        kf_std_x.append(sx)

        [(odom_x[i]-true_x[i])**2 + (odom_y[i]-true_y[i])**2

    kf_err = np.sqrt(np.mean(
        [(kf_x_list[i]-true_x[i])**2 + (kf_y_list[i]-true_y[i])**2
         for i in range(n)]
    ))

    print(f"\nKalman Filter Sensor Fusion Results:")
    print(f"  Odometry-only RMSE:     {odom_err*100:.1f} cm")
    print(f"  Kalman Filter RMSE:     {kf_err*100:.1f} cm")

    ax1.plot(odom_x, odom_y,     'r--', lw=1.5, alpha=0.7,

    ax1.scatter(uwb_x_list, uwb_y_list, s=30, c='green', alpha=0.5,
                zorder=4, label='UWB measurements')
    ax1.plot(kf_x_list, kf_y_list, 'b-', lw=2,

    ax1.set_xlabel('x (m)');  ax1.set_ylabel('y (m)')
    ax1.set_title('Sensor Fusion: Odometry + UWB via Kalman Filter')
    ax1.legend(fontsize=9);  ax1.grid(True, alpha=0.3);  ax1.set_aspect('equal')

    ax2 = axes[1]
    ax2.plot(kf_std_x, 'b-', lw=1.5, label='KF position std (m)')

        self.R_uwb = np.diag([0.04, 0.04])   # sigma = 0.2 m each axis

        # History for plotting
        self.hist = [(x0, y0)]

    # ── Predict step ──────────────────────────────────────────
    def predict(self, d_left_m, d_right_m, wheel_base_m):
        """
        EKF Predict step using wheel encoder readings.

        Parameters:
            d_left_m:     Distance traveled by left wheel this timestep (m)
            d_right_m:    Distance traveled by right wheel this timestep (m)
            wheel_base_m: Distance between left and right wheels (m)
        """
        d      = (d_right_m + d_left_m) / 2.0
        dtheta = (d_right_m - d_left_m) / wheel_base_m

        x_val, y_val, theta = (float(self.x[0]),
                                float(self.x[1]),
                                float(self.x[2]))

        mid_theta = theta + dtheta / 2.0

        # ── Nonlinear motion model f(x, u) ──
        x_new     = x_val + d * np.sin(mid_theta)
        y_new     = y_val + d * np.cos(mid_theta)
        theta_new = theta + dtheta

        # Wrap theta to [-pi, pi]
        theta_new = (theta_new + np.pi) % (2*np.pi) - np.pi

        self.x = np.array([[x_new], [y_new], [theta_new]])

        # ── Jacobian F = df/dx ──
        # Partial derivatives of each state component with respect to [x, y, theta]
        # f1 = x + d*sin(theta + dtheta/2) → df1/dtheta = d*cos(mid_theta)
        # f2 = y + d*cos(theta + dtheta/2) → df2/dtheta = -d*sin(mid_theta)
        # f3 = theta + dtheta              → df3/dtheta = 1
        F = np.array([
            [1, 0,  d * np.cos(mid_theta)],   # ∂x_new/∂[x, y, theta]
            [0, 1, -d * np.sin(mid_theta)],   # ∂y_new/∂[x, y, theta]
            [0, 0,  1                    ]    # ∂theta_new/∂[x, y, theta]
        ])

        # ── Covariance prediction ──
        self.P = F @ self.P @ F.T + self.Q

        self.hist.append((float(self.x[0]), float(self.x[1])))

    # ── Update step — UWB ─────────────────────────────────────
    def update_uwb(self, meas_x, meas_y):
        """
        EKF Update step with UWB position measurement.
        Returns Kalman gain diagonal (for monitoring).
        """
        z = np.array([[meas_x], [meas_y]])

        # Observation matrix H — UWB measures x and y directly (linear!)
        H = np.array([[1, 0, 0],
                      [0, 1, 0]])

        # Innovation: difference between measurement and prediction
        innovation = z - H @ self.x

        # Innovation covariance
        S = H @ self.P @ H.T + self.R_uwb

        # Kalman Gain
        K = self.P @ H.T @ np.linalg.inv(S)

        # State update
        self.x = self.x + K @ innovation

        # Covariance update — Joseph form (numerically stable)
        I_KH   = np.eye(3) - K @ H
        self.P = I_KH @ self.P @ I_KH.T + K @ self.R_uwb @ K.T

        return float(K[0, 0]), float(K[1, 1])

    # ── Update step — LiDAR ICP ───────────────────────────────
    def update_lidar_icp(self, icp_dx, icp_dy, icp_dtheta,
                          icp_cov=None):
        """
        EKF Update with LiDAR ICP pose correction.
        ICP gives a relative correction to the current pose estimate.

        Parameters:
            icp_dx, icp_dy, icp_dtheta: ICP-estimated correction
            icp_cov: 3×3 ICP result covariance (defaults to conservative estimate)
        """
        if icp_cov is None:
            icp_cov = np.diag([0.02, 0.02, 0.005])

        # ICP corrects the state directly — H is identity
        H   = np.eye(3)
        z   = self.x + np.array([[icp_dx], [icp_dy], [icp_dtheta]])

        S   = H @ self.P @ H.T + icp_cov
        K   = self.P @ H.T @ np.linalg.inv(S)

        self.x += K @ (z - H @ self.x)

        I_KH   = np.eye(3) - K @ H
        self.P = I_KH @ self.P @ I_KH.T + K @ icp_cov @ K.T

    def get_pose(self):
        return float(self.x[0]), float(self.x[1]), float(self.x[2])

    def get_position_std(self):
        return float(np.sqrt(self.P[0,0])), float(np.sqrt(self.P[1,1]))


# ── Full simulation ────────────────────────────────────────
def run_ekf_simulation():
    np.random.seed(7)

    WHEEL_BASE  = 0.20
    WHEEL_DIAM  = 0.07
    TPR         = 360
    UWB_EVERY_N = 8

    ekf = RobotEKF()

    # Build commands for a partial square + curve
    def ticks(d_m):
        return int(d_m * TPR / (np.pi * WHEEL_DIAM))

    def turn_ticks():
        return int((WHEEL_BASE/2) * (np.pi/2) * TPR / (np.pi * WHEEL_DIAM))

    commands = []
    for _ in range(25):  commands.append((ticks(0.05), ticks(0.05)))  # straight
    for _ in range(10):  commands.append((turn_ticks()//10, -(turn_ticks()//10)))  # right turn
    for _ in range(25):  commands.append((ticks(0.05), ticks(0.05)))  # straight
    for _ in range(10):  commands.append((turn_ticks()//10, -(turn_ticks()//10)))  # right turn
    for _ in range(15):  commands.append((ticks(0.05), ticks(0.05)))  # partial straight

    # True path (cumulative, with small noise)
    true_path = [(0.0, 0.0)]
    tx, ty, tt = 0.0, 0.0, 0.0
    for (dL, dR) in commands:
        true_dL = dL*(np.pi*WHEEL_DIAM)/TPR + np.random.normal(0, 0.001)
        true_dR = dR*(np.pi*WHEEL_DIAM)/TPR + np.random.normal(0, 0.001)
        d  = (true_dR + true_dL)/2
        dt = (true_dR - true_dL)/WHEEL_BASE
        mid = tt + dt/2
        tx += d*np.sin(mid);  ty += d*np.cos(mid);  tt += dt
        true_path.append((tx, ty))

    # Odometry (noisier)
    odom_path = [(0.0, 0.0)]
    ox, oy, ot = 0.0, 0.0, 0.0
    for (dL, dR) in commands:
        odL = dL*(np.pi*WHEEL_DIAM)/TPR + np.random.normal(0, 0.004)
        odR = dR*(np.pi*WHEEL_DIAM)/TPR + np.random.normal(0, 0.004)
        d  = (odR + odL)/2
        dt = (odR - odL)/WHEEL_BASE
        mid = ot + dt/2
        ox += d*np.sin(mid);  oy += d*np.cos(mid);  ot += dt
        odom_path.append((ox, oy))

    # EKF with UWB updates
    ekf2 = RobotEKF()
    for step, (dL, dR) in enumerate(commands):
        dL_m = dL*(np.pi*WHEEL_DIAM)/TPR + np.random.normal(0, 0.004)
        dR_m = dR*(np.pi*WHEEL_DIAM)/TPR + np.random.normal(0, 0.004)
        ekf2.predict(dL_m, dR_m, WHEEL_BASE)

        if step % UWB_EVERY_N == 0 and step < len(true_path):
            true_step = true_path[min(step+1, len(true_path)-1)]
            ux = true_step[0] + np.random.normal(0, 0.07)
            uy = true_step[1] + np.random.normal(0, 0.07)
            ekf2.update_uwb(ux, uy)

    # Compute RMSE
    n = min(len(odom_path), len(true_path), len(ekf2.hist))
    odom_rmse = np.sqrt(np.mean(
        [(odom_path[i][0]-true_path[i][0])**2+(odom_path[i][1]-true_path[i][1])**2
         for i in range(n)]))
    ekf_rmse = np.sqrt(np.mean(
        [(ekf2.hist[i][0]-true_path[i][0])**2+(ekf2.hist[i][1]-true_path[i][1])**2
         for i in range(n)]))

    print(f"EKF Simulation Results:")
    print(f"  Odometry RMSE:   {odom_rmse*100:.1f} cm")
    print(f"  EKF RMSE:        {ekf_rmse*100:.1f} cm")
    print(f"  Improvement:     {(1-ekf_rmse/odom_rmse)*100:.0f}%")

    fig, ax = plt.subplots(figsize=(9, 8))
    tx_list = [p[0] for p in true_path]
    ty_list = [p[1] for p in true_path]
    ox_list = [p[0] for p in odom_path]
    oy_list = [p[1] for p in odom_path]
    ex_list = [p[0] for p in ekf2.hist]
    ey_list = [p[1] for p in ekf2.hist]

    ax.plot(tx_list, ty_list, 'k-',  lw=2.5, label='True path', zorder=5)
    ax.plot(ox_list, oy_list, 'r--', lw=1.5, alpha=0.7,
            label=f'Odometry only (RMSE={odom_rmse*100:.1f}cm)')
    ax.plot(ex_list, ey_list, 'b-',  lw=2,
            label=f'EKF fusion (RMSE={ekf_rmse*100:.1f}cm)')
    ax.plot(0, 0, 'go', ms=12, label='Start', zorder=10)

    ax.set_xlabel('x (m)');  ax.set_ylabel('y (m)')
    ax.set_title('EKF Robot Localization\nOdometry (encoders) + UWB Fusion\n'
                 '(Vertical-axis θ convention — Robotics & Intelligence Nigeria Ring)')
    ax.legend(fontsize=10);  ax.grid(True, alpha=0.3);  ax.set_aspect('equal')
    plt.tight_layout()
    plt.savefig('ekf_localization.png', dpi=150)
    plt.show()


if __name__ == '__main__':
    run_ekf_simulation()
