WK 01
Course Intro & Modern Computing Landscape
⏱ ~12 min read
⌄
Architecture Foundations

📖 4-Hour Summary

Modern computing is hitting physical walls. Dennard Scaling — the rule that shrinking transistors kept power density constant — broke around 2004. Today, processors can't simply clock faster without melting. This created the power wall and the phenomenon of dark silicon: chips where large portions must stay powered off to prevent overheating. The memory wall is a separate bottleneck — CPUs process data far faster than memory can supply it. These constraints forced the industry toward heterogeneous and specialised computing. This course explores how the field responded.

🔑 Key Concepts to Know

  • Dennard Scaling: As transistors shrink, power density stays constant. Broke ~2004 — now smaller = hotter.
  • Dark Silicon: Only a fraction of a chip can operate at full power simultaneously due to thermal limits.
  • Memory Wall: CPU speed scales faster than DRAM bandwidth — data starvation is the real bottleneck.
  • Amdahl's Law: Speedup from parallelism is limited by the serial portion of a program.

WK 02
Advances in Computer Architecture I
⏱ ~15 min read
⌄
Architecture

📖 4-Hour Summary

Modern processor design has evolved dramatically post-Dennard. Chiplet architectures break monolithic chips into smaller dies connected via high-speed fabric (AMD's Infinity Fabric, Intel's EMIB), improving yield and enabling heterogeneous integration. 3D stacking (HBM memory stacked on logic) dramatically cuts the memory wall by bringing DRAM within millimetres of the processor. RISC-V, an open-source ISA, is disrupting proprietary processor design — it's now used in embedded, server, and AI chips worldwide. The key insight: modern advances are about packaging, interconnects, and openness, not just raw transistor counts.

🔑 Key Concepts to Know

  • Chiplets: Small dies manufactured separately and integrated into one package — better yield, heterogeneous mix.
  • HBM (High Bandwidth Memory): DRAM stacked vertically in a package with the processor, massive bandwidth improvement.
  • RISC-V: Open ISA — anyone can build a compliant processor without licensing fees. Disruptive to ARM/x86.
  • Processing-in-Memory (PIM): Moving compute logic into/near memory arrays to eliminate data movement bottlenecks.

WK 03
Advances in Computer Architecture II
⏱ ~15 min read
⌄
Architecture

📖 4-Hour Summary

This week deepens architectural analysis. The Roofline Model is a critical tool — it plots achieved performance against arithmetic intensity to reveal whether a workload is compute-bound or memory-bandwidth-bound. Out-of-order execution and speculative execution are the two mechanisms that extract ILP (Instruction-Level Parallelism) from sequential code — but they also create security vulnerabilities (covered in Week 8). Domain-specific architectures (DSAs) like Google's TPU show that custom silicon designed for one workload class can beat general-purpose CPUs by 100x in performance-per-watt. The "New Golden Age" thesis: specialisation is the path forward.

🔑 Key Concepts to Know

  • Roofline Model: Visual tool — x-axis = arithmetic intensity (FLOP/byte), y-axis = performance. Reveals bottleneck.
  • Speculative Execution: CPU predicts branches and executes ahead — rolls back on wrong prediction.
  • Domain-Specific Architecture: Custom hardware for one task class (e.g., TPU for matrix multiply) — major efficiency gain.
  • ILP (Instruction-Level Parallelism): Executing multiple independent instructions simultaneously within a single core.

▸ Pillar II — Accelerators & Edge (Wks 4–7)
WK 04
Heterogeneous Computing & Accelerators
⏱ ~15 min read
⌄
Architecture AI/ML

📖 4-Hour Summary

Heterogeneous computing means combining different processor types (CPU + GPU + FPGA + ASIC) in one system, each handling the workloads it's best suited for. GPUs use SIMT (Single Instruction Multiple Threads) execution — thousands of simple cores execute the same instruction on different data simultaneously. This is ideal for matrix operations. FPGAs are reprogrammable hardware — you define logic circuits at runtime. Microsoft Catapult used FPGAs to accelerate Bing search ranking at data-center scale. ASICs are fixed-function chips — maximum efficiency, zero flexibility. The programmer's job is to correctly map workload characteristics to the right hardware substrate.

🔑 Key Concepts to Know

  • SIMT: GPU model — threads grouped in warps (32), all execute the same instruction. Divergence kills performance.
  • Memory Coalescing: GPU threads in a warp accessing contiguous memory are served in one transaction — critical for performance.
  • FPGA (Field-Programmable Gate Array): Reconfigurable logic fabric — can be reprogrammed for different algorithms.
  • Systolic Array: Grid of processors passing data in rhythm — used in Google TPU for matrix multiply.

WK 05
GPU Architecture & Deep Learning Hardware
⏱ ~15 min read
⌄
Architecture AI/ML

📖 4-Hour Summary

Modern AI workloads are dominated by matrix multiplications and attention mechanisms. Google's TPU (Tensor Processing Unit) uses a systolic array to multiply matrices at 100+ TOPS efficiency. NVIDIA's Tensor Cores (introduced in Volta) perform mixed-precision (FP16/BF16/INT8) matrix multiply in a single clock — enabling the current LLM training revolution. The memory bandwidth problem is central here: training large transformers is bottlenecked by how fast weights can be loaded, not compute throughput. NVLink and NVSwitch create high-bandwidth GPU interconnects to treat multiple GPUs as one logical accelerator. Knowing this explains why H100 clusters cost what they do.

🔑 Key Concepts to Know

  • Tensor Core: Specialised compute unit in NVIDIA GPUs — performs D = A×B + C in mixed precision per clock.
  • TPU Systolic Array: 2D grid of MAC units — data flows through like a wave, no memory access during compute.
  • Flash Attention: Memory-efficient attention algorithm — reduces HBM reads by computing in SRAM tiles.
  • NVLink: NVIDIA's high-bandwidth GPU-to-GPU interconnect — up to 900 GB/s bidirectional in H100 SXM.

WK 06
Edge Computing & Embedded Systems
⏱ ~14 min read
⌄
Edge/IoT AI/ML

📖 4-Hour Summary

Edge computing moves computation from the cloud to devices near data sources — reducing latency, bandwidth costs, and privacy risk. The motivation: a self-driving car cannot afford 100ms round-trips to the cloud. TinyML deploys quantised neural networks on microcontrollers (ARM Cortex-M) consuming milliwatts. Federated Learning trains models across distributed edge devices without centralising raw data — each device trains locally, only gradient updates are shared. Key challenge: edge devices are resource-constrained (limited RAM, no GPU, battery-powered). Solutions involve model compression: quantisation, pruning, knowledge distillation, and hardware-aware neural architecture search.

🔑 Key Concepts to Know

  • TinyML: Deploying ML models on microcontrollers — INT8 quantised models under 1MB, sub-mW inference.
  • Federated Learning: Train locally, share only gradients — preserves data privacy, reduces bandwidth.
  • Quantisation: Reducing weight precision (FP32 → INT8) — 4x memory reduction with minimal accuracy loss.
  • Fog Computing: Intermediate layer between edge and cloud — aggregates edge data before cloud offload.

WK 07
Internet of Things (IoT) Systems Design
⏱ ~14 min read
⌄
Edge/IoT Systems

📖 4-Hour Summary

IoT systems connect physical sensors and actuators to the digital world at massive scale. Two dominant protocols define IoT communication: MQTT (publish-subscribe, low overhead, TCP-based — used for smart home, industrial sensors) and CoAP (request-response, UDP-based — designed for constrained devices, similar to HTTP but lightweight). The Matter protocol (backed by Apple, Google, Amazon) is the new unified smart-home standard. IoT systems face the "triple constraint": low power, low cost, and wireless connectivity. LoRaWAN enables long-range, ultra-low-power sensor networks. Security is the critical unsolved challenge — most deployed IoT devices have no update mechanism and run outdated firmware.

🔑 Key Concepts to Know

  • MQTT: Lightweight pub/sub protocol for IoT — broker-based, minimal overhead, runs over TCP/443.
  • CoAP: Constrained Application Protocol — RESTful, UDP-based, for devices too small for HTTP.
  • LoRaWAN: Long-Range Wide Area Network — km-range, sub-mW operation, ideal for agricultural/industrial sensors.
  • Matter: Universal smart-home connectivity standard — runs over IP (Wi-Fi/Thread), supported by all major platforms.

▸ Pillar III — Security & Real-Time (Wks 8–10)
WK 08
Hardware Security & Trusted Computing
⏱ ~16 min read
⌄
Security Architecture

📖 4-Hour Summary

Spectre exploits speculative execution: the CPU predicts a branch, speculatively executes code that shouldn't run, and although the result is rolled back, the cache state is not — an attacker can measure cache timing to infer what data was accessed across privilege boundaries. Meltdown collapsed kernel-user isolation entirely via out-of-order execution. Mitigations (KPTI, Retpoline) impose 5–30% performance penalties. Intel SGX (Software Guard Extensions) creates hardware-encrypted "enclaves" — isolated regions where even the OS cannot read your code/data. TPM 2.0 provides a hardware root of trust: secure key storage, platform attestation, and sealed secrets that unlock only on a specific, verified platform configuration.

🔑 Key Concepts to Know

  • Spectre: Exploits speculative execution — poisons branch predictor to leak cross-privilege data via cache side-channel.
  • Meltdown: Out-of-order execution reads kernel memory before privilege check — cache encodes the secret.
  • Flush+Reload: Side-channel technique — flush a cache line, wait for victim, time reload to infer victim's access.
  • SGX Enclave: Encrypted, isolated code+data region — CPU verifies integrity, OS/hypervisor cannot inspect.
  • TPM 2.0: Trusted Platform Module — hardware chip for key storage, measured boot, and remote attestation.

WK 09
Secure Enclaves & Side-Channel Attacks
⏱ ~15 min read
⌄
Security

📖 4-Hour Summary

Side-channel attacks extract secrets by measuring physical or temporal by-products of computation — not the computation itself. Power analysis (Simple/Differential) correlates power consumption with cryptographic operations to extract AES keys from smart cards. EM (Electromagnetic) attacks use near-field probes — effective even without physical contact with the device. Rowhammer is a DRAM vulnerability: repeatedly accessing a row causes bit flips in adjacent rows due to electrical crosstalk — this can flip a security-critical bit (e.g., a page table entry) without ever "accessing" the target. SGX has been broken by Foreshadow, Plundervolt, and SGAxe — demonstrating that hardware security requires constant adversarial evaluation.

🔑 Key Concepts to Know

  • Differential Power Analysis (DPA): Statistical analysis of power traces — extracts crypto keys from embedded devices.
  • Rowhammer: Bit flips in DRAM from repeated row access — can escalate privileges or escape sandboxes.
  • Foreshadow: L1 terminal fault — breaks SGX enclave isolation using speculative execution on page-not-present entries.
  • Countermeasures: Masking (randomise power profile), shielding (EM), ECC memory (Rowhammer), microcode patches (Spectre class).

WK 10
Real-Time Systems & RTOS
⏱ ~15 min read
⌄
Real-Time Systems

📖 4-Hour Summary

Real-time systems must respond to events within strict time deadlines — missing a deadline is a system failure, not just poor performance. Hard real-time systems (aircraft flight control, ABS brakes) tolerate zero deadline misses. Soft real-time (video streaming) tolerates occasional misses. The Liu & Layland (1973) paper established Rate Monotonic Scheduling (RMS): assign higher priority to higher-frequency tasks. RMS is optimal for fixed-priority scheduling and is provably schedulable if CPU utilisation ≤ 69%. EDF (Earliest Deadline First) is optimal for dynamic priority but harder to implement. FreeRTOS and Zephyr are dominant open-source RTOS platforms for embedded real-time systems.

🔑 Key Concepts to Know

  • Hard vs Soft Real-Time: Hard = deadline miss = system failure. Soft = deadline miss = degraded quality.
  • Rate Monotonic Scheduling (RMS): Higher frequency = higher priority. Optimal fixed-priority. Bound: U ≤ n(2^(1/n) - 1) → ~69%.
  • EDF: Earliest Deadline First — dynamically assigns priority. Optimal — can utilise 100% CPU.
  • Priority Inversion: Low-priority task holds resource needed by high-priority task. Solution: Priority Inheritance Protocol.

WK 11
Quantum Computing Fundamentals
⏱ ~16 min read
⌄
Quantum

📖 4-Hour Summary

Classical bits are 0 or 1. Qubits exist in superposition — a weighted combination of both until measured. Entanglement correlates qubits such that measuring one instantly determines the other — useful for certain algorithms but not for faster-than-light communication. Grover's Algorithm provides quadratic speedup (O(√N) vs O(N)) for searching unsorted databases — practical for cryptographic brute-force resistance evaluation. Shor's Algorithm factors integers in polynomial time — it would break RSA-2048 with a large fault-tolerant quantum computer. Today's NISQ devices (50–1000 noisy qubits) can't run Shor at cryptographic scale yet. Quantum error correction requires ~1000 physical qubits per logical qubit — we need millions of physical qubits for real impact.

🔑 Key Concepts to Know

  • Superposition: Qubit = α|0⟩ + β|1⟩ where |α|² + |β|² = 1. Measurement collapses to 0 or 1 probabilistically.
  • Grover's Algorithm: O(√N) search — quadratic speedup. Doubles effective key length against brute-force.
  • Shor's Algorithm: Polynomial-time factorisation — threatens RSA/ECC. Requires fault-tolerant QC to execute.
  • Decoherence: Environmental noise destroys quantum state. T1/T2 times measure how long qubits survive.
  • NISQ: Noisy Intermediate-Scale Quantum — current era. VQE and QAOA are near-term practical algorithms.

WK 12
Neuromorphic & Brain-Inspired Computing
⏱ ~14 min read
⌄
Neuromorphic AI/ML

📖 4-Hour Summary

The human brain computes at ~20 watts — far more efficiently than any silicon system. Neuromorphic computing mimics biological neural computation using Spiking Neural Networks (SNNs): neurons fire discrete spikes in time, not continuous activations. This temporal sparsity means most neurons are idle most of the time — huge energy savings. Intel Loihi 2 implements 1M neurons on-chip with programmable learning rules — it learns in real time without a training phase. IBM TrueNorth has 4096 cores emulating 1M neurons at 65mW. The key challenge: training SNNs is hard because spike functions are non-differentiable (can't use backpropagation directly). Surrogate gradient methods and ANN-to-SNN conversion are current solutions.

🔑 Key Concepts to Know

  • Spiking Neural Network (SNN): Neurons communicate via discrete spike events — sparse, event-driven, low energy.
  • Spike-Timing-Dependent Plasticity (STDP): Biological learning rule — synaptic strength depends on relative spike timing of pre/post neurons.
  • Intel Loihi 2: 1M neurons, on-chip learning, programmable. Used for robotics, sparse coding, optimisation problems.
  • Memristors: Resistive memory devices that can store synaptic weights in analog — key for in-memory SNN compute.

▸ Pillar V & VI — AI Systems & Autonomous Engineering (Wks 13–16)
WK 13
AI/ML System Integration
⏱ ~15 min read
⌄
AI/ML Architecture

📖 4-Hour Summary

Deploying AI is a systems engineering problem, not just a modelling problem. LLM inference is memory-bandwidth bound — the KV-cache for large context windows demands hundreds of GB of HBM. Quantisation (INT4/INT8) reduces memory footprint 4–8x with < 1% accuracy loss using techniques like GPTQ and AWQ. Model pruning removes redundant weights — structured pruning creates hardware-friendly sparse patterns; unstructured pruning achieves higher sparsity but needs sparse compute support. AI compilers (TVM, MLIR, XLA) bridge high-level model graphs to hardware-specific instruction sets — operator fusion, tiling, and layout optimisation are key techniques. LoRA (Low-Rank Adaptation) enables fine-tuning of billion-parameter models on a single GPU by updating only a small rank-decomposition matrix.

🔑 Key Concepts to Know

  • KV-Cache: Stores key/value projections from past tokens — avoids recomputation, but grows linearly with context length.
  • Quantisation (INT4/INT8): Maps FP32 weights to low-bit integers — reduces memory, increases throughput, minimal quality loss.
  • Operator Fusion: Compiler combines adjacent ops (e.g., MatMul + ReLU) into one kernel — eliminates intermediate memory reads.
  • LoRA: Fine-tuning via W = W₀ + BA where B,A are low-rank matrices — trains 0.1% of parameters.

WK 14
Autonomous Systems & Cyber-Physical Systems
⏱ ~15 min read
⌄
Systems Real-Time

📖 4-Hour Summary

Cyber-Physical Systems (CPS) integrate computation, communication, and physical processes — autonomous vehicles, surgical robots, smart grids. The key challenge: physical processes are continuous and uncertain; digital systems are discrete and deterministic. ROS 2 (Robot Operating System) provides middleware for autonomous systems with DDS-based real-time communication. AUTOSAR is the automotive equivalent — a layered software architecture for vehicle ECUs. Time-Sensitive Networking (TSN) extends Ethernet with deterministic timing guarantees — critical for control loops that can't tolerate jitter. Digital twins create real-time simulation counterparts of physical systems, enabling predictive maintenance and "what-if" scenario testing without physical risk. Formal verification ensures software correctness before deployment in safety-critical systems.

🔑 Key Concepts to Know

  • CPS: Tight integration of computation with physical dynamics — correctness depends on timing, not just logic.
  • ROS 2 + DDS: Robot middleware — nodes communicate via publish/subscribe over Data Distribution Service (real-time capable).
  • TSN (IEEE 802.1): Deterministic Ethernet — time-aware shapers guarantee maximum latency for control traffic.
  • Digital Twin: Live simulation counterpart of a physical system — fed by real sensor data, used for monitoring and optimisation.

WK 15
5G, Network Slicing & Systems Integration
⏱ ~13 min read
⌄
Systems Edge/IoT

📖 4-Hour Summary

5G is not just faster 4G — it's a new architecture. The key innovations: mmWave spectrum (24–100 GHz) for extreme bandwidth but short range; Massive MIMO (hundreds of antennas) for spatial multiplexing; and ultra-low latency (< 1ms air interface) enabling real-time control applications. Network Slicing uses software-defined networking (SDN) and network function virtualisation (NFV) to carve logically isolated networks from shared physical infrastructure — a slice for IoT sensors, a slice for autonomous vehicles, a slice for broadband, each with different QoS guarantees. The O-RAN Alliance is driving open, disaggregated radio access networks — separating radio hardware from software, enabling AI-driven network management. This connects 5G directly to edge computing: the base station becomes an edge compute node.

🔑 Key Concepts to Know

  • 5G NR Three Pillars: eMBB (enhanced Mobile Broadband), URLLC (Ultra-Reliable Low-Latency), mMTC (Massive Machine Type Comms).
  • Network Slicing: Virtual isolated networks over shared infrastructure — each with tailored SLA, QoS, and isolation.
  • Massive MIMO: 64–256 antenna arrays — beamforming to individual users, dramatically increases spectral efficiency.
  • O-RAN: Open Radio Access Network — disaggregates RU/DU/CU, enables AI-driven xApp controllers for optimisation.
